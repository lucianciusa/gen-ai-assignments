Hybrid Search Playbook

Hybrid search combines vector similarity and lexical matching. This often performs better than vector-only or keyword-only approaches in enterprise content with mixed terminology.

When to Use Hybrid Search

  • Users write natural language questions with domain-specific terms.
  • Documents include exact identifiers such as product IDs or policy names.
  • You need a balance between semantic understanding and precise term matches.

Query Patterns

  • Vector-only: best for conceptual similarity.
  • Keyword-only: best for exact term recall.
  • Hybrid: recommended default for practical retrieval systems.
  • Semantic hybrid: stronger ranking quality when semantic configuration is mature.

Evaluation Checklist

  1. Prepare 10 to 20 representative prompts.
  2. Capture top-5 results for each search mode.
  3. Mark relevance manually and compare averages.
  4. Use findings to tune field mappings and ranking options.
