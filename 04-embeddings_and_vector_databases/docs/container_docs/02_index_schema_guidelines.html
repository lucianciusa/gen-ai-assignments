Index Schema Design Guidelines

A well-designed schema improves both retrieval quality and maintainability. Field definitions should align with expected query behavior, filtering needs, and semantic ranking configuration.

Field Categories

  • Key field: unique document identifier.
  • Title field: concise document or section title.
  • Content field: main text used for keyword and semantic relevance.
  • Metadata fields: source, category, language, and publication date.
  • Vector field: embeddings generated from content or chunks.

Configuration Advice

  • Mark content as searchable and retrievable.
  • Use filterable metadata for category and document-type constraints.
  • Keep field names consistent and descriptive.
  • Match semantic configuration fields with your most informative text.

Common Mistakes

  • Storing huge mixed-content blobs in a single field without chunking.
  • Forgetting to include metadata fields needed for filtering.
  • Using title fields that are too generic, such as "Document".
  • Not validating retrieval quality after index creation.
