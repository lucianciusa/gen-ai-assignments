# Final Practice: Build a SaaS Application with Generative AI

This final practice consists of building a **complete SaaS (Software as a Service) application** that integrates **Generative AI** processes. Applying everything learned in the course, you must create a full-stack app around a completely free theme that could be viable in a real market (as if launching a startup).

**Important restriction:** AI agents are **NOT allowed**. The app must be based on Generative AI processes (text generation, image generation, embeddings, RAG, etc.) but not multi-agent systems.

**Work mode:** Teams of 2 or 3.

**Time available:** 4-5 days.

---

## Project Description

Your team must conceptualize, design, and build a complete SaaS app that solves a real problem and has business potential. The app must:

- **Be full-stack:** include both functional backend and frontend.
- **Integrate Generative AI:** include at least one Generative AI process that provides real value (not just a basic chatbot).
- **Have a SaaS mindset:** designed as a service real users could consume.
- **Be commercially viable:** the idea should make business sense and solve a real market need.

### Example ideas (inspiration only):
- Personalized social media content generator
- Technical/academic writing assistant with RAG
- Sentiment analysis platform for customer feedback
- Design/mockup generator from text descriptions
- Personalized recommendation system with embeddings
- CV and cover-letter optimization tool
- Adaptive educational content generation platform

The topic is fully open, but it must show practical Generative AI application.

---

## Grading Criteria

Project evaluation is distributed as follows:

- **Technical (75%)**: code quality, architecture, AI implementation, technical robustness, scalability, technical documentation.
- **Business (25%)**: business viability, sales pitch, value proposition, market potential.

---

## Project Presentation

Each team must deliver a full presentation including:

### 1. Sales Presentation (10 minutes)
- **Business pitch:** present your product as if seeking investment or customers.
- **Problem and solution:** what problem does your app solve?
- **Value proposition:** why would someone pay for your service?
- **Target market:** who is it for?
- **Business model:** how would it generate revenue?
- **Differentiation:** what makes your solution unique?

### 2. Live Demo (5 minutes)
- **Functional demo:** show main features working.
- **User flow:** walk through a real end-to-end use case.
- **AI integration:** clearly demonstrate how AI adds value.

**Total presentation time:** 15 minutes per team.

---

## Deliverables

### 1. GitHub Repository

You must submit a public GitHub repository including:

#### Complete README.md with:
- **Project description:** what does the app do?
- **Technologies used:** stack (frameworks, libraries, cloud services, AI models, etc.)
- **Technical architecture:** diagram or system architecture explanation
- **AI integration description:** what Generative AI process is used and how?
- **Main features:** list of implemented capabilities
- **Local deployment tutorial:** step-by-step instructions to run locally (open-source style)
  - Prerequisites (Node.js, Python, environment variables, etc.)
  - Dependency installation
  - Configuration (API keys, databases, etc.)
  - Commands to run backend and frontend
  - Access URLs
- **Authors:** team member names

#### Source code:
- **Organized and structured:** clear folders for backend, frontend, config, etc.
- **Documented:** readable code with comments where needed
- **Best practices:** follow conventions for the chosen language/framework

#### Other files:
- Appropriate `.gitignore`
- `requirements.txt` or `package.json` for dependencies
- `.env.example` listing required environment variables (no real credentials)

### 2. Presentation

- **Sales deck:** PDF or PowerPoint

---

## Minimum Requirements (Mandatory)

For the project to be valid, it must include all of the following:

1. **Functional Full-Stack App:**
   - Working backend with functional endpoints/services
   - Frontend with usable user interface
   - Correct integration between frontend and backend

2. **Generative AI Integration:**
   - At least one clear Generative AI process (text generation, embeddings, RAG, fine-tuning, image generation, etc.)
   - AI must provide real value to the app, not decorative usage
   - Agent-based systems are prohibited

3. **Complete GitHub Repository:**
   - Detailed technical README
   - Functional and verified local deployment tutorial
   - Complete executable source code

4. **Complete Presentation:**
   - Sales pitch (10 min)
   - Live functional demo (5 min)
